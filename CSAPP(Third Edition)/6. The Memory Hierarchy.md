# 6.1 Storage Technologies

## 6.1.1 Random Access Memory

- 随机访问存储器(Random-Access Memory, RAM)分为两类：静态(SRAM)和动态(DRAM)
- SRAM将每个位存储在一个双稳态(bistable)的存储器单元里
- DRAM将每个位存储为对一个电容的充电
- DRAM芯片中的单元（位）被分成d个超单元(supercell)，每个超单元都由w个DRAM单元组成，一个d x w的DRAM总共存储了dw位信息。超单元被组织成一个r行c列的长方形阵列，r x c = d。每个超单元有(i, j)的地址，i表示行，j表示列
- 每个DRAM芯片被连接到某个称为内存控制器（memory controller）的电路，它可以一次传入/出 w 位到每个DRAM芯片
	- 内存控制器先将行地址i（RAS(Row Access Strobe)）发送到DRAM，然后是列抵制j（CAS(Column Access Strobe)）
	- DRAM把超单元(i, j)内容发给内存控制器作为回应
	- RAS和CAS请求共享相同的DRAM地址引脚
- DRAM芯片封装在内存模块（memory module）中，插在主板扩展槽上
- 存储在ROM(Read-Only Memory)设备中的程序通常被称为固件(firmware)
- 总线事务(bus transaction) = 读事务(read transaction) 主存->CPU + 写事务(write transaction) CPU->主存

## 6.1.2 Disk Storage

- 磁盘操作对扇区的访问时间主要有三个主要部分：寻道时间，旋转时间和传送时间
- 磁盘控制器：维护逻辑块号和实际物理磁盘扇区之间的映射关系
- CPU使用内存映射I/O（memory- mapped I/O）向I/O设备发射命令
- 设备可以自己执行读或者写总线事务而不需要CPU干涉的过程，称为直接内存访问（Direct Memory Access，DMA）这种数据传送称为DMA传送（DMA transfer）

## 6.1.3 Solid State Disks

- 固态硬盘（Solid State Disks）由一个或多个闪存芯片和闪存翻译层（flash translation layer）组成

## 6.1.4 Storage Technology Trends

- 内存和磁盘技术的基本事实：增加密度（从而降低成本）比降低访问时间容易的多

# 6.2 Locality

- 局部性原理(principle of locality)：良好的计算机程序具有良好的局部性，即倾向于引用临近于其他最近引用过的数据项的数据项，或者最近引用过的数据项本身。
- 局部性通常有两种不同的形式：时间局部性（temporal locality）和空间局部性（spatial locality）
	- 	时间局部性：被引用过一次的内存位置很可能在不远的将来再被多次引用
	-  空间局部性：一个内存被引用了一次，那么程序很可能在不远的将来引用附近的一个内存位置

## 6.2.3 Summary of Locality

- 重复引用相同变量的程序有良好的时间局部性
- 对于步长为k的引用模式的程序，步长越小，空间局部性越好
- 对于取指令来说，循环有好的时间和空间局部性

# 6.3 The Memory Hierarchy

## 6.3.1 Caching in the Memory Hierarchy 

- 数据总是以块大小为传送单元（transfer unit）在相邻的存储器两层之间来回复制
- 缓存命中(cache hit)
- 缓存不命中(cache miss)
	- 覆盖一个现存块的过程称为替换(replacing)或驱逐(evicting)
	- 被驱逐的块被称作牺牲块(victim block)
	- 决定该替换哪个块是由缓存的替换策略(replacement policy)来控制

| 替换策略 | 英文 | 核心思想 | 优缺点 |
|-----|------|---------|--------|
| **随机** | Random | 随便挑一个踢掉 | 实现简单，但效果不稳定 |
| **LRU** | Least Recently Used | 踢掉最久没用过的 | 效果好，但需要追踪访问时间 |
| **LFU** | Least Frequently Used | 踢掉访问次数最少的 | 适合热点数据，但新数据容易被误杀 |
| **FIFO** | First In First Out | 最早进来的先踢 | 简单，但不考虑使用频率 |
| **Clock/Second Chance** | — | LRU 的近似，用一个 bit 标记 | 硬件常用，平衡性能和效果 |

- 缓存不命中的种类
	- 空缓存(cold cache)，强制性不命中(compulsory miss) 或冷不命中(cold miss)
	- 只要发生了不命中，缓存就必须执行某个放置策略(placement policy)
	- 限制性的放置策略会引起冲突不命中(conflict miss)
	- 阶段性访问缓存块的某个相对稳定不变的集合称为工作集(working set)，当工作集的大小超过缓存大小，缓存会经历容量不命中(capacity miss)

| 放置策略 | 灵活性 | 查找成本 | 实际应用 |
|-----|-------|---------|---------|
| Direct Mapped | 最低 | 最快 | 大容量 cache |
| N-way Set Associative | 中等 | 中等 | L1/L2 常用（4-way, 8-way） |
| Fully Associative | 最高 | 最慢 | 只用于很小的 cache（如 TLB） |

## 6.3.2 Summary of Memory Hierarchy Concepts

- 利用时间局部性：Replacement Policy
- 利用空间局部性：Block Transfer
